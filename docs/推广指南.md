# FlowDesk 推广指南

## 一、推广前准备

### 1. 制作演示 GIF / 短视频

这是最重要的推广素材，所有平台都需要。

**工具推荐：**
- [ScreenToGif](https://www.screentogif.com/) — 免费，录制屏幕为 GIF
- OBS Studio — 免费，录制 MP4 视频

**录制内容建议（30-60秒）：**
1. 打开 FlowDesk，展示深色主题界面
2. 拖拽几个节点到画布（触发节点 + 操作节点）
3. 连线，配置节点参数
4. 点击运行，展示执行过程和日志
5. 展示执行结果（比如文件被自动移动/复制）

**注意：** 操作要流畅，不要拖沓。GIF 控制在 5MB 以内。

---

### 2. 优化 GitHub 项目页面

打开你的仓库，编辑 `README.md`，替换为以下内容：

```markdown
# FlowDesk

桌面自动化工作流引擎 — 用拖拽的方式构建自动化任务。

![demo](docs/demo.gif)  <!-- 替换为你的 GIF 路径 -->

## 功能

- 可视化画布编辑器，拖拽节点、连线即可构建工作流
- 多种触发方式：手动执行、定时任务(Cron)、文件监听、全局热键
- 丰富的节点类型：
  - 操作：剪贴板、数据库、延时、邮件、文件操作、HTTP请求、系统通知、命令执行、数据转换
  - 逻辑：条件判断、循环、重试、异常捕获、变量、子工作流调用
- 工作流管理：创建、重命名、复制、删除、批量操作
- 执行日志：完整的工作流执行历史记录
- 系统托盘：最小化到托盘，快速运行工作流
- 深色主题 UI

## 下载

前往 [Releases](https://github.com/你的用户名/flowdesk/releases) 下载最新版本安装包。

## 技术栈

- Electron + React + TypeScript
- React Flow (画布编辑器)
- Zustand (状态管理)
- better-sqlite3 (本地数据库)

## 开发

npm install
npm run dev

## 构建安装包

npm run package:win

## 许可证

MIT
```

把录制好的 GIF 放到 `docs/demo.gif`，然后推送更新。

### 3. 部署介绍页面到 GitHub Pages

你已有 `docs/FlowDesk介绍.html`，可以免费部署为网站。

**步骤：**

1. 打开 GitHub 仓库页面 → **Settings** → **Pages**
2. **Source** 选择 `Deploy from a branch`
3. **Branch** 选择 `main`，文件夹选择 `/docs`
4. 点 **Save**
5. 等待 1-2 分钟，访问 `https://你的用户名.github.io/flowdesk/`

部署成功后，把这个链接作为软件的官方主页，所有推广都指向这里。

## 四、持续运营

1. **及时回复评论** — 所有平台的评论和反馈都要回复
2. **收集用户需求** — 记录用户想要的功能，排入开发计划
3. **定期更新版本** — 每次更新都发布新的 Release，在各平台同步更新帖
4. **建立交流群** — 用户多了之后建 QQ群或微信群，方便交流

---

## 五、注意事项

- 所有帖子/文章都要附上下载链接（GitHub Release 或 GitHub Pages）
- 截图和 GIF 是最重要的素材，没有图的帖子效果差很多
- 不要一次发太多平台，容易顾不过来，优先 V2EX + 少数派 + B站
- 保持耐心，推广是一个持续的过程
